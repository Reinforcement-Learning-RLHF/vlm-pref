# Rollout Format Specification


## Folder Structure

Each rollout is stored as its own subfolder under `data/rollouts/`:

```
data/
└── rollouts/
    ├── <rollout_id>/
    │   ├── metadata.json 
    │   ├── <video_filename>  # e.g. rollout_agentview.mp4
    │   └── any other data  # e.g. arm action at each timestep in h5py or any other data form
    └── <rollout_id>/
        ├── metadata.json
        ├── <video_filename>
        └── any other data 
```

---

## metadata.json Fields

Each rollout folder must contain a `metadata.json` file with the following fields:

| Field | Type | Description |
|---|---|---|
| `rollout_id` | string | Unique identifier for this rollout. Should match the folder name. |
| `task_name` | string | Short machine-readable task name, e.g. `pour`, `lift`, `pick_place`. |
| `task_goal` | string | Human-readable description of what a successful rollout looks like. |
| `video_filename` | string | Filename of the video inside this rollout folder, e.g. `rollout_agentview.mp4`. |
| `video_path` | string | Path to the video relative to the project root, e.g. `data/rollouts/pour_good_clean_001/rollout_agentview.mp4`. |
| `simulator` | string | Simulator used, e.g. `robosuite`, `isaac`, `mujoco`. |
| `success_label` | boolean | `true` if the rollout completed the task successfully, `false` otherwise. |
| `timestamp` | string | ISO 8601 datetime when the rollout was recorded, e.g. `2026-05-30T14:00:00Z`. |
| `additional_notes` | string | Free-text field for any extra context (failure mode, anomalies, param notes, etc.). |

### Example

```json
{
  "rollout_id": "pour_good_clean_001",
  "task_name": "pour",
  "task_goal": "Tilt the container to pour its contents into the target cup without spilling.",
  "video_filename": "rollout_agentview.mp4",
  "video_path": "data/rollouts/pour_good_clean_001/rollout_agentview.mp4",
  "simulator": "robosuite",
  "success_label": true,
  "timestamp": "2026-05-30T14:00:00Z",
  "additional_notes": "Smooth, controlled pour at 110 deg tilt. Speed 1.35."
}
```

## Extra notes

- The `rollout_id` in `metadata.json` must match the folder name exactly.
- If a rollout has multiple camera views (e.g. `agentview` and `wristview`), create one subfolder per view, each with its own `metadata.json` and video file.
- The `video_path` field should use forward slashes and be relative to the project root.
- `success_label` must be a JSON boolean (`true` / `false`), not a string.
- If a field is not applicable, omit it rather than using `null`; the registry script treats missing optional fields as empty strings.

## rollout_registry.csv

All rollouts are indexed in `data/rollout_registry.csv`. This file is auto-generated by `scripts/create_rollout_registry.py` and should not be edited by hand. It contains one row per rollout with the columns listed above.
